Semantic elements = elements with a meaning.

Examples of non-semantic elements: <div> and <span> - Tells nothing about its content.

Examples of semantic elements: <form>, <table>, and <article> - Clearly defines its content.

		<section> element: 

*Defines a section in a document. Here are some examples of where can be used: 

-Chapters
-Introduction
-News items
-Contact information

		<article> element:

Specifies independent, self-contained content. Here are some examples of where can be used: 

-forum posts
-blog posts
-user comments
-product cards 
-newspaper articles

		<header> element: 

represents a container for introductory content or set of navigational links. Typically contains: 

-One or more heading elements (<h1> - <h6>)
-Logo or icon 
-authorship information

<header> CANNOT BE PLACED WITHIN A <footer>, <address> OR ANOTHER <header> element. 

		<footer> element: 

Defines a footer for a document or section. Typically contains: 

-authorship information 
-copuright information 
-contact information 
-sitemap
-back to top links 
-related documents 

		<nav> element:

Defines a set of navigation links. 
inside this elemen goes the following example ( <a href="link">HTML<\a> )

		<aside> element: 

Defines some content aside from the content it is placed in (like a sidebar). Should be indirectly related to the surrounding content. 

	<figure> and <figcaption> elements:

<figure> tag specifies self-contained content, like illustrations, diagrams, photos, code listings, etc. 

<figcaption> tag defines a caption for a <figure> element. The <figcaption> element can be placed as the first or as the last child of a <figure> element.
